2. iOS调用说明

接口名称：BliBaoPaySDK
接口描述：提供支付功能
BliBaoPay接口主要为商户提供订单支付功能。接口所提供的方法，如下表所示：

方法名称 方法描述
- (void)startPay:(NSDictionary*)params :(void(^)(NSDictionary*resultDic))result; 开始请求支付
- (void)blibaoPayProcessOrderWithPaymentResult:(NSURL*)url; 处理支付宝返回的url
便利宝支付接口调用说明

方法原型

- (void)startPay:(NSDictionary*)params :(void(^)(NSDictionary*resultDic))result;

方法功能:给商户提供快捷支付功能

参数名称 参数描述
params 支付请求参数字典
resultDic 支付返回结果信息
付:请求便利宝授权参数说明
参数名称 参数描述
app_auth_token 银盒子分配给调用商户授权,如:(Yeu63u362u373781ieyee)
app_id 银盒子分配给调用商ID,如:(2014072300007148)
key MD5密钥银盒子提供
method 支付宝支付:blibao.alipay.app.pay
out_order_no 商户网站唯一订单号.如:(70501111111S001111119)
passback_params @”orderNum:123456”(扩展参数非必填)
subject 商品的标题/交易标题/订单标题/订单关键字等,如:(大乐透)
timestamp 订单生成时间
total_amount 订单价格
notify_url 支付回调地址
pay_id 微信支付是从微信开放平台获取的app_id
支付回调接口说明

方法原型

- (void)blibaoPayProcessOrderWithPaymentResult:(NSURL*)url ;

方法功能:处理支付宝客户端返回的URL

参数名称 参数描述
url 支付宝客户端回传的url
3. 支付返回字段说明
返回结果参数
字段名 说明 示例
code 结果码 0000
msg 结果描述 支付成功
result 结果详情 “{@”out_trade_no”:@”55555”,
@”total_amount”:@”0.01”,
@”timestamp”:@”2017-03-21 12:25:20”,
@”trade_no”:@”70501111111S001111119”}
result说明
字段名 说明 示例
out_trade_no 商家订单号 70501111111S001111119
total_amount 订单金额 0.02
timestamp 便利宝订单创建时间 2017-03-21 12:25:20
trade_no 支付宝交易流水号 4549473434643434343
order_num 便利宝订单号 70501111111S001111119
code说明
返回码 含义
0000 网络连接错误
9000 订单支付成功
8000 正在处理中，支付结果未知（有可能已经支付成功），请查询商户订单列表中订单的支付状态
4000 订单支付失败
5000 重复请求
6001 用户中途取消
6002 网络连接出错
6004 支付结果未知（有可能已经支付成功），请查询商户订单列表中订单的支付状态
10000 接口调用成功，调用结果请参考具体的API文档所对应的业务返回参数
20000 服务不可用
20001 授权权限不足
40001 缺少必选参数
40002 非法的参数
40004 业务处理失败
40006 权限不足
